DistributedSupervisor

Inherits from: Object
Instance variables: strategy, childSpecs, maxRestarts, maxSeconds, restartTimes, running, cluster, supervisorProcess, children, nodeAssignments, rrIndex

A distributed supervisor that manages child processes across cluster nodes. Extends the Supervisor pattern to spawn children on remote nodes via spawnOn:, monitor them with cross-node monitors, and handle node failures by restarting affected children on surviving nodes.

Children are assigned to nodes either explicitly (via node on the spec) or automatically via round-robin placement across cluster members.

Test 
| ds <DistributedSupervisor> |
ds := DistributedSupervisor new: #oneForOne children: #() cluster: nil.
ds strategy >>> #oneForOne
Test 
| ds <DistributedSupervisor> |
ds := DistributedSupervisor new: #oneForAll children: #() cluster: nil maxRestarts: 5 maxSeconds: 10.
ds maxRestarts >>> 5
Example 
"Create a distributed supervisor across a 2-node cluster"
| cluster sup spec1 spec2 |
cluster := Cluster seeds: #('worker1:8081' 'worker2:8081').
cluster start.

spec1 := ChildSpec id: 'db' start: [DbPool new run].
spec2 := ChildSpec id: 'web' start: [WebServer new run] restart: #transient.

sup := DistributedSupervisor new: #oneForOne children: { spec1. spec2 } cluster: cluster.
sup start.

"Children are distributed across cluster members via round-robin"
sup runningChildren.
sup countChildren.

sup stop.
cluster stop
