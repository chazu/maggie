# gowrap Example

Demonstrates wrapping a Go standard library package (`strings`) for use from Maggie code.

## What This Example Does

Wraps Go's `strings` package and calls package-level functions (`Contains`, `HasPrefix`, `HasSuffix`, `ToUpper`, `ToLower`) as class methods on `Go::Strings`.

## Workflow

### 1. Wrap the Go package

```bash
cd examples/gowrap
mag wrap
```

This reads `maggie.toml`, introspects the `strings` package, and generates:
- `wrap/strings/wrap.go` -- Go glue code with primitive registrations
- `wrap/strings/stubs.mag` -- Maggie stub classes for the compiler

### 2. Build the binary

```bash
mag build -o example-app
```

Compiles `src/Main.mag` into a VM image, embeds it in a Go binary alongside the wrapped `strings` primitives, and produces `./example-app`.

### 3. Run

```bash
./example-app
```

Expected output:

```
contains Maggie: true
contains Python: false
hasPrefix Hello: true
hasSuffix !: true
toUpper hello: HELLO
toLower HELLO: hello
```

## Project Structure

```
examples/gowrap/
  maggie.toml          # Project manifest with [go-wrap] config
  go.mod               # Go module (needed for mag build)
  src/
    Main.mag           # Maggie source using wrapped Go functions
  wrap/                # Generated by mag wrap (gitignored)
    strings/
      wrap.go          # Go glue code
      stubs.mag        # Maggie stub classes
```

## How It Works

The `[go-wrap]` section in `maggie.toml` declares which Go packages to wrap:

```toml
[[go-wrap.packages]]
import = "strings"
include = ["Contains", "HasPrefix", "HasSuffix", "ToUpper", "ToLower"]
```

The stubs directory must be included in `[source].dirs` so the compiler knows about the wrapped classes.

At build time:
1. `mag wrap` introspects the Go package and generates glue code
2. `mag build` compiles Maggie sources (including stubs) into a VM image
3. The image and Go glue are compiled into a standalone binary
4. On startup, the binary loads the image, registers Go primitives, and runs `Main.start`

Go functions become class methods on a namespace class (e.g., `Go::Strings`). Use the fully-qualified name `Go::Strings` in Maggie source.

## Known Limitations

- The `[go-wrap]` output directory must be set to a simple path like `wrap` (not `.maggie/wrap`) due to a filepath.Base issue in BuildEmbedded
- Wrapped Go struct types (like `strings.Builder`) register under `Go::Strings::Builder` in Go glue but `Go::Builder` in stubs -- this naming mismatch prevents using wrapped struct methods
- Only package-level functions with basic Go types (string, int, float, bool, []byte) are wrapped; complex types are silently skipped
